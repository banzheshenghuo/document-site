## 概述

本次更新为文档站添加了首页 Tab 分类功能，实现了精读文章与收藏文章的视觉区分，并优化了 Skill 文档的文件命名规范。

## ✅ 主要更新

### 1. 首页 Tab 分类功能

**新增功能**：
- 📊 全部 Tab：显示所有文章
- 📚 精读 Tab：只显示精读文章（`[精读]`、`[解读]`、`[深度]` 前缀）
- ⭐ 收藏 Tab：只显示收藏文章（`[收藏]`、`[归档]`、`[书签]` 前缀）
- 📅 今天 Tab：只显示 24 小时内创建的文章

**技术实现**：
- 通过文件名前缀自动分类
- 使用 JavaScript 实现客户端 Tab 切换
- 动态显示/隐藏对应类型的文档卡片

**用户体验**：
- 一键切换不同类型的文章
- 每个 Tab 显示该类型的文章数量
- 空状态时显示对应的提示信息

### 2. 文章命名前缀规则

**精读文章**：
```bash
[精读] 文章标题.md
[解读] 文章标题.md
[深度] 文章标题.md
```

**收藏文章**：
```bash
[收藏] 文章标题.md
[归档] 文章标题.md
[书签] 文章标题.md
```

**优势**：
1. **文件管理**：快速识别文章类型
2. **首页分类**：文档站通过前缀自动分类
3. **视觉区分**：文件列表中一目了然
4. **避免冲突**：避免不同类型文章使用相同标题

## 📝 知识点总结

### 1. 前缀命名的重要性

**为什么需要前缀**：
1. **自动分类**：文档站通过前缀自动分类到不同 Tab
2. **快速识别**：文件列表中一目了然
3. **避免冲突**：不同类型文章可以同名
4. **便于管理**：快速筛选和搜索特定类型文章

**前缀规范**：
- `[精读]` - 精读文章
- `[收藏]` - 收藏文章
- `[归档]` - 归档文章
- `[书签]` - 书签文章
- `[解读]` - 解读文章
- `[深度]` - 深度解读文章

### 2. 日期格式的重要性

**为什么需要双引号**：
1. **YAML 规范**：字符串类型值必须用引号包裹
2. **避免错误**：缺少引号会导致构建失败
3. **类型安全**：确保日期被正确识别为字符串
4. **兼容性**：符合 YAML 标准格式

**正确格式**：
```yaml
date: "2026-04-01 05:18:00"
```

### 3. Tab 分类的优势

**用户体验**：
- 一键切换查看不同类型文章
- 减少滚动和查找时间
- 提升信息获取效率

**开发效率**：
- 自动化分类，减少手动整理
- 前缀命名规范便于管理
- 清晰的文档结构
