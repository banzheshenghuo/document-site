## 问题 2：标签前缀显示问题

### 文件名确认

两篇文章的文件名已经包含了正确的前缀：

```bash
# 精读文章
[精读] 技术人该积累什么才能避免被AI淘汰-深度解读.md

# 收藏文章
[收藏] 微软开源AI量化交易神器Qlib-收藏归档.md
```

### 标签前缀说明

- **精读文章**：`[精读]` 开头
- **收藏文章**：`[收藏]` 开头
- **格式正确**：前缀和标题之间有空格

### 可能的原因

如果你在文档站首页没有看到标签前缀，可能是以下原因：

1. **浏览器缓存**：浏览器缓存了旧版本的页面
2. **文档站缓存**：GitHub Pages 可能还在部署新版本
3. **没有刷新**：页面没有完全刷新

### 验证步骤

1. **强制刷新浏览器**：
   - Windows/Linux: `Ctrl + Shift + R`
   - macOS: `Cmd + Shift + R`

2. **清除浏览器缓存**：
   - 打开开发者工具
   - 右键点击刷新按钮
   - 选择"清空缓存并硬性重新加载"

3. **检查部署状态**：
   - 访问 GitHub Actions 页面
   - 查看最新的部署是否成功
   - 等待几分钟让部署完成

4. **直接访问文章页面**：
   - 点击文章卡片进入详情页
   - 查看 URL 中的文章标题
   - 确认是否包含前缀
