**检查结果**：
两篇文章的文件名已经正确包含了前缀：

- ⭐ 收藏 Tab：应该只显示 `[收藏]` 开头的文章
- 📅 今天 Tab：应该只显示当天创建的文章

## 解决方案

### 1. 文件命名规范

**正确的格式**：
```bash
# 精读文章
[精读] 文章标题.md
[解读] 文章标题.md
[深度] 文章标题.md

# 收藏文章
[收藏] 文章标题.md
[归档] 文章标题.md
[书签] 文章标题.md
```

**当前状态**：
两篇文章的文件名已经是正确的格式，包含前缀。

### 2. 首页 Tab 分类

**功能说明**：
- 📚 精读 Tab：只显示包含 `[精读]`、`[解读]`、`[深度]` 前缀的文章
- ⭐ 收藏 Tab：只显示包含 `[收藏]`、`[归档]`、`[书签]` 前缀的文章
- 📅 今天 Tab：只显示 24 小时内创建的文章
- 📊 全部 Tab：显示所有文章

**验证步骤**：
1. 访问文档站首页
2. 点击 📚 精读 Tab
3. 查看是否显示 "技术人该积累什么才能避免被AI淘汰-深度解读" 这篇文章
4. 点击 ⭐ 收藏 Tab
5. 查看是否显示 "微软开源AI量化交易神器Qlib-收藏归档" 这篇文章

### 3. 技能实现细节

**前端代码**（src/pages/index.astro）：
```javascript
// 分类逻辑
const readDocs = sortedDocs.filter(doc => {
  const title = doc.data.title || ''
  return title.includes('[精读]') || title.includes('[解读]')
})

const collectDocs = sortedDocs.filter(doc => {
  const title = doc.data.title || ''
  return title.includes('[收藏]') || title.includes('[归档]') || title.includes('[书签]')
})
```

**标题显示**：
DocCard 组件会直接显示完整的 `title`，包括前缀。
例如：`[收藏] 微软开源AI量化交易神器Qlib`

### 4. 可能的问题

**问题 1**：前缀可能显示不突出
- 当前前缀和标题都是同样的字体和颜色
- 用户可能期望前缀有更明显的视觉区分

**建议优化**：
1. 为前缀添加不同的颜色（红色表示精读，橙色表示收藏）
2. 为前缀添加 Badge 标签
3. 在标题中单独显示前缀标签

**问题 2**：筛选逻辑可能有问题
- 需要验证 Tab 切换是否正常工作
- 确认空状态消息是否正确
