## 问题分析

根据截图，文章标题中没有显示前缀（`[精读]` 或 `[收藏]`）。

**原因**：
DocCard 组件直接显示完整的 `title` 字段，虽然标题中包含前缀，但在视觉上没有明显区分，导致用户看不清。

---

## 解决方案

### 方案选择

我选择了**方案 1**：分离前缀和标题，添加样式区分

**为什么不选方案 2**：
方案 2（颜色区分）需要为每个类型单独定义颜色，代码较复杂，且颜色可能不统一。

---

## 实现内容

### 1. DocCard 组件优化

**添加的功能**：
- **前缀提取函数**：从标题中提取前缀和标题
  ```javascript
  function getPrefix(title: string) {
    const match = title.match(/^\[([^\]]+)\](.*)$/)
    return match ? { prefix: match[1], titleWithoutPrefix: match[2] } : { prefix: null, titleWithoutPrefix: title }
  }
  ```

- **前缀类型识别**：识别前缀属于精读还是收藏
  ```javascript
  function getPrefixType(prefix: string | null): 'read' | 'collect' | null {
    if (!prefix) return null
    if (prefix === '精读' || prefix === '解读' || prefix === '深度') return 'read'
    if (prefix === '收藏' || prefix === '归档' || prefix === '书签') return 'collect'
    return null
  }
  ```

- **样式类**：为卡片添加类型特定的类名
  ```javascript
  const prefixType = getPrefixType(prefix)
  const className = `doc-card ${prefixType ? `doc-card--${prefixType}` : ''}`
  ```
